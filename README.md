# Trailing Period URL Encoding — Proof of Concept

This is a small PHP app that empirically tests browser behaviour around
percent-encoded trailing periods in URLs, built in the context of
[Wikimedia Phabricator task T423306](https://phabricator.wikimedia.org/T423306).

## The problem

Wikipedia article titles can end with a period — e.g.
[Mata v. Avianca, Inc.](https://en.wikipedia.org/wiki/Mata_v._Avianca,_Inc.)

When that URL is shared as plain text in a markdown-aware platform (Phabricator,
GitHub, Lemmy, etc.), the linkifier treats the trailing period as
sentence-ending punctuation and strips it, producing a broken link:

```
See https://en.wikipedia.org/wiki/Mata_v._Avianca,_Inc.
                                                       ^--- stripped, link breaks
```

## The proposed fix

MediaWiki's `wfUrlencode()` function (and its JavaScript counterpart
`mw.internalWikiUrlencode()`) should percent-encode a trailing period as
`%2E`, producing:

```
https://en.wikipedia.org/wiki/Mata_v._Avianca,_Inc%2E
```

Markdown linkifiers see no trailing period to strip, so the link survives.

## A question worth investigating

A commenter on T423306 wondered whether this fix would actually hold up in
practice: even if the server generates `%2E`, browsers decode it to `.` in
the address bar — so would a user copying and sharing the URL end up with a
bare period again anyway?

## What this POC shows

This app serves pages under `/wiki/<title>` with no redirects, and displays
exactly what `REQUEST_URI` the server received. It has three test links:

| Link | `href` attribute |
|---|---|
| Bare period | `/wiki/Mata_v._Avianca,_Inc.` |
| Percent-encoded | `/wiki/Mata_v._Avianca,_Inc%2E` |
| No trailing period | `/wiki/Regular_Article` |

**Key finding:** when clicking the percent-encoded link, Firefox:

- Displays `.` in the address bar (normalising for readability)
- **Copies `%2E` to the clipboard** when the URL is copied

The fix holds up. Firefox stores and copies the URL in its encoded form; the
decoded display is cosmetic only. A URL generated by MediaWiki with `%2E`
will survive a copy-paste into a markdown context intact.
